Инструкция по использованию POST Info v1.4
=======================


Для чего эта программа?
-----------------------
Расшифровывает POST-коды и звуковые коды
BIOS в обычный текст с помощью БД.

Каковы системные требования PostInfo?
-----------------------
Windows 95 Rus (но лучше если Win98);
16Mb ОЗУ (желательно 32);
60 Kb на винчестере (даже меньше);
Visual Basic 6 Runtime;
ComDlg32.ocx;
POST-карта и не менее 1 глаза для узнавания POST-кодов;
1+ внимательное ухо для звуковых кодов;

Я ввёл код/звук, что дальше делать?
-----------------------
Справа от полей ввода находится кнопка.
Нажмите соответствующую кнопку.

Какие BIOS поддерживаются?
-----------------------
Теоретический любые. Однако предустановлен
только один файл описания
BIOS (ФОБ) - для AwardBIOS v4.51/6.0PG. Ожидается
выпуск файлов для PhoenixBIOS и AMI, но вот когда...

Как загрузить другой файл описания BIOS?
-----------------------
Нажмите кнопку "Выбрать BIOS".

Как сохранить результат запроса в файл?
-----------------------
Нажмите кнопку "Сохранить".

Каков формат файлов описания BIOS'ов?
-----------------------
INI. Формат очень понятный, даже описывать не надо.
Открывается в "Блокноте". Примечание: если нужно
вставить перенос в описание POST'а или Beep'а,
надо там ввести " ПЕРЕНОС " (без кавычек и с
пробелами). В качестве примера рассмотрите
файл "award.ini".

Чего стало нового в версии 1.4.1?
-----------------------
Во-первых, исправлены некоторые баги. Во-вторых,
внесена поддержка изменения размера окна сей софтины.
В-третьих, улучшена навигация с клавиатуры. Теперь не
нужно перебирать контролы TAB-ом. Просто ввёл, и
жмякнул "Enter". Также удалена реклама сайта "ROM.By",
к которому я никакого отношения не имею, он даже не
платил за рекламу. Просто хороший сайт про BIOS и
диагностику POST.
А так, больших изменений посравнению с 1.4.0 нет.

Если ругается на ошибку Ini.dll
-----------------------
Скопируйте файл Ini.dll из архива с программой в каталог
%WINDIR%\System32 и совершите от имени аминистратора
"RegSvr32 C:\Windows\System32\Ini.dll"
в штатной коммандной строке Windows.
